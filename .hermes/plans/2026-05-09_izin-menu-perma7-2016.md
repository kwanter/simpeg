# Izin Menu: Izin Keluar Kantor, Izin Pulang Cepat, Izin Tidak Masuk Kerja

Created: 2026-05-09 14:50 WITA
Repo: `/Users/macbook/Developer/php/simpeg`
Mode: planning → implementation

> **For Hermes:** Use subagent-driven-development skill to implement task-by-task.

**Goal:** Add three dedicated izin menus to Simpeg matching PERMA No. 7 Tahun 2016:
1. **Izin Keluar Kantor** — same-day, time-range based (jam_mulai → jam_selesai), single-level approval
2. **Izin Pulang Cepat** — same-day, time-range based, same rules as Izin Keluar Kantor
3. **Izin Tidak Masuk Kerja** — date range, max 2 work days, two-level approval (atasan → pimpinan)

**Architecture:**
- Reuse existing `Izin` model, `IzinController`, and `IzinPolicy` — add jenis_izin enum values and jenis-specific logic in controller/policy
- Add three Blade view pairs (create + index) per izin type, sharing the same route/controller but with jenis pre-selected and type-specific form constraints
- Fix existing PDF bugs (`$izin->atasan` → `$izin->atasan_pimpinan`)
- Create separate PDF templates per izin type following Lampiran II (keluar/pulang cepat) and Lampiran III (tidak masuk)
- Navigation: three new menu items visible to roles with `create izin` + `view izin` permissions

**Tech Stack:** Laravel 10, PHP 8.1+, Blade, Spatie Permission, barryvdh/laravel-dompdf, WorkdayService, UUID primary keys

```php
private function validateIzinKeluarKantor(Request $request): array
{
    $rules = [
        'tanggal_mulai' => ['required', 'date', 'date_equals:' . now()->toDateString()],
        'tanggal_selesai' => ['required', 'date', 'date_equals:' . now()->toDateString()],
        'jam_mulai' => ['required', 'date_format:H:i'],
        'jam_selesai' => ['required', 'date_format:H:i', 'after:jam_mulai'],
        'alasan' => ['required', 'string', 'max:500'],
    ];
    $messages = [
        'tanggal_mulai.date_equals' => 'Izin keluar kantor hanya dapat diajukan pada hari ini.',
        'tanggal_selesai.date_equals' => 'Izin keluar kantor hanya dapat diajukan pada hari ini.',
        'jam_selesai.after' => 'Jam selesai harus setelah jam mulai.',
    ];
    return [$rules, $messages];
}

/**
 * Validasi khusus untuk Izin Tidak Masuk Kerja.
 * Pasal 8 PERMA No. 7 Tahun 2016 — Lampiran III — Maks 2 hari kerja
 */
private function validateIzinTidakMasukKerja(Request $request): array
{
    $rules = [
        'tanggal_mulai' => ['required', 'date', 'after_or_equal:today'],
        'tanggal_selesai' => ['required', 'date', 'after_or_equal:tanggal_mulai'],
        'alasan' => ['required', 'string', 'max:500'],
    ];
    $messages = [
        'tanggal_selesai.after_or_equal' => 'Tanggal selesai harus sama atau setelah tanggal mulai.',
    ];
    return [$rules, $messages];
}
```

```php
public function verifyPimpinan(Izin $izin): bool
{
    if (in_array($izin->jenis_izin, ['Izin Keluar Kantor', 'Izin Pulang Cepat'])) {
        return false; // Single-level approval — no pimpinan step
    }
    $user = auth()->user();
    if (!$user) return false;
    if ($user->hasRole(['super-admin', 'admin'])) return true;
    if (!$user->hasRole('pimpinan')) return false;
    if ($izin->verifikasi_atasan !== 'Disetujui') return false;
    if ($izin->verifikasi_pimpinan !== 'Belum Diverifikasi') return false;

}
```

```php
public function generatePdf(Izin $izin): Response
{
    $this->authorize('cetak', $izin);

    // Select PDF template based on jenis izin
    $template = match ($izin->jenis_izin) {
        'Izin Keluar Kantor', 'Izin Pulang Cepat' => 'izin.pdf-keluar-kantor',
        'Izin Tidak Masuk Kerja' => 'izin.pdf-tidak-masuk',
        default => 'izin.pdf',
    };

    $pdf = \PDF::loadView($template, ['izin' => $izin]);
    $nama = Str::slug($izin->pegawai->nama ?? 'Pegawai');
    $noSurat = $izin->no_surat_izin ?? $izin->uuid;
    return $pdf->download("Surat_Izin_{$nama}_{$noSurat}.pdf");
}
```

```php
/**
 * Index filtered for Izin Keluar Kantor & Pulang Cepat.
 * Single-level approval menu.

public function indexKeluarKantor(): View
{
    $this->authorize('viewAny', Izin::class);
    
    $user = auth()->user();
    $query = Izin::with(['pegawai', 'atasan_pimpinan', 'pimpinan'])
        ->whereIn('jenis_izin', ['Izin Keluar Kantor', 'Izin Pulang Cepat']);

    if (!$user->hasRole(['super-admin', 'admin', 'atasan-pimpinan', 'pimpinan', 'verifikator'])) {
        $pegawai = Pegawai::where('nip', $user->nip)->first();
        $query->where('pegawai_uuid', $pegawai?->uuid);
    }

    $izins = $query->latest()->paginate(20);
    return view('izin.index-keluar-kantor', compact('izins'));
}
```

## Open Questions Before Execution

1. **Navigation UX**: Should "Izin Keluar Kantor" and "Izin Tidak Masuk" be separate top-level menus, or sub-items under "Pengajuan Izin"? The plan above uses sub-items.
2. **Izin Pulang Cepat**: Treat as the same as "Izin Keluar Kantor" (same Lampiran II, same rules), or should it be a separate dedicated page?
3. **Document upload**: PERMA allows optional documents. Should upload be enabled for new izin types?
4. **PDF signing**: The Lampiran forms have signature lines. Should we add uploaded signature images or keep text-only?
5. **Jam kerja validation**: Should `jam_mulai` and `jam_selesai` be validated against Pasal 4 jam kerja (08:00-16:30 / 08:00-17:00)? Plan currently allows any time.
